Capitulo 1 - 7

  • git init: detecta los archivos
  • git add (nombre archivo): indica a cual archivo queremos hacerle seguimiento
  • git add . : agrega todos los archivos a la fase de staging
  • git commit -m "": dentro de las comillas ponemos un nombre a la copia que queremos hacer con el comando git commit
  • git reset . :baja ese archivo de la fase de staging
  • git push: actualiza de local a nube
  • git pull: actualiza de nube a local
  • git branch (nombre rama): esto crea una rama
  • git checkout (nombre rama): esto nos mueve a la rama que ingresemos
  • git checkout -- . : Esto regresa nuestro archivo a como estaba en el ultimo commit
  • git branch -d (nombre de la rama): esto borra la rama que queramos
  • cuando hay un conflicto, visual studio code nos propone 4 opciones para solucionar el problema
  • cuando escojemos nuestra solucion al problema, nos queda el archivo en estado de conflicto, para arreglar esto hay que hacer un git add y un commit
